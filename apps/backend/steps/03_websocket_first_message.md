# Step 3 -- WebSocket Connection and First 1:1 Message

**Goal:** Two users connect over WebSocket and exchange a text message. The message is persisted to Postgres.

**What this step is NOT:** This is a naive, minimal implementation. There is no hub pattern, no Redis pub/sub fan-out, no offline queue, no authentication middleware. All of that comes later. The only purpose of this step is to prove the plumbing works -- two WebSocket clients can talk to each other through the server.

**Prerequisites (from steps 1-2):**
- HTTP server running on `:8080`
- User registration endpoint working
- Postgres with `users` table, migrations via tern
- Redis client connected

---

## 1. Install gorilla/websocket

```bash
go get github.com/gorilla/websocket
```

This is the de-facto WebSocket library for Go. It handles the HTTP upgrade handshake and gives us a `*websocket.Conn` to read/write frames.

**`internal/model/message.go`**

```go
package model

import (
	"encoding/json"
	"time"

	"github.com/google/uuid"
)

// DeliveryState tracks how far a message has progressed.
type DeliveryState int16

const (
	StatePending   DeliveryState = 0 // created but not yet sent over WS
	StateSent      DeliveryState = 1 // delivered to recipient's WS conn
	StateDelivered DeliveryState = 2 // recipient acked receipt (future step)
	StateRead      DeliveryState = 3 // recipient opened / read it (future step)
)

// Message is the core chat message stored in Postgres.
type Message struct {
	ID          uuid.UUID    `json:"id"`
	ClientID    uuid.UUID    `json:"client_id"`    // idempotency key set by the sender
	SenderID    uuid.UUID    `json:"sender_id"`
	RecipientID uuid.UUID    `json:"recipient_id"`
	GroupID     *uuid.UUID   `json:"group_id,omitempty"` // nil for 1:1
	Body        string       `json:"body"`
	State       DeliveryState `json:"state"`
	CreatedAt   time.Time    `json:"created_at"`
	UpdatedAt   time.Time    `json:"updated_at"`
}

// Envelope is the top-level frame sent over the WebSocket.
// Type tells the handler what kind of payload to expect.
//
//	"message"      -> payload is a Message
//	"ack"          -> payload is an ACK
//	"state_update" -> payload is a StateUpdate
//	"read"         -> payload is a StateUpdate with State=Read
type Envelope struct {
	Type    string          `json:"type"`
	Payload json.RawMessage `json:"payload"`
}

// ACK is sent back to the sender after a message is persisted.
type ACK struct {
	MessageID uuid.UUID `json:"message_id"`
}

// StateUpdate is used to move a message through delivery states.
type StateUpdate struct {
	MessageID uuid.UUID     `json:"message_id"`
	ClientID  uuid.UUID     `json:"client_id"`
	State     DeliveryState `json:"state"`
}
```

Key decisions:
- `ClientID` is a UUID generated by the sending client. It acts as an idempotency key so duplicate sends (e.g. from a retry) do not create duplicate rows.
- `GroupID` is a pointer so it can be `nil` for 1:1 messages. We will use it in a later step.
- `Envelope` uses `json.RawMessage` so we can defer parsing the payload until we know the type.

## 3. Messages migration -- `internal/database/migrations/002_create_messages.sql`

Tern picks up files in alphabetical order from the embedded `migrations/` directory.

**`internal/database/migrations/002_create_messages.sql`**

```sql
CREATE TABLE messages (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    client_id    UUID UNIQUE NOT NULL,
    sender_id    UUID NOT NULL REFERENCES users(id),
    recipient_id UUID REFERENCES users(id),
    group_id     UUID,
    body         TEXT NOT NULL,
    state        SMALLINT NOT NULL DEFAULT 0,
    created_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at   TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_messages_sender    ON messages(sender_id);
CREATE INDEX idx_messages_recipient ON messages(recipient_id);
CREATE INDEX idx_messages_client_id ON messages(client_id);
---- create above / drop below ----

DROP TABLE IF EXISTS messages;
```

Notes:
- `client_id UUID UNIQUE` enforces idempotency at the DB level.
- `sender_id` has a foreign key to `users(id)`. `recipient_id` does too but is nullable for future group messages.
- `state SMALLINT` maps directly to our `DeliveryState` enum in Go.

## 4. Message repository -- `internal/repository/message.go`

```bash
mkdir -p internal/repository
```

**`internal/repository/message.go`**

```go
package repository

import (
	"context"
	"fmt"

	"github.com/google/uuid"
	"github.com/jackc/pgx/v5"
	"github.com/jackc/pgx/v5/pgxpool"

	"github.com/Rengoku1926/wp_proto/apps/backend/internal/model"
)

type MessageRepo struct {
	pool *pgxpool.Pool
}

func NewMessageRepo(pool *pgxpool.Pool) *MessageRepo {
	return &MessageRepo{pool: pool}
}

// Create inserts a message with idempotency on client_id.
// If a row with the same client_id already exists the insert is
// silently skipped and the existing row is returned.
func (r *MessageRepo) Create(ctx context.Context, msg *model.Message) (*model.Message, error) {
	query := `
		INSERT INTO messages (client_id, sender_id, recipient_id, group_id, body, state)
		VALUES ($1, $2, $3, $4, $5, $6)
		ON CONFLICT (client_id) DO NOTHING
		RETURNING id, client_id, sender_id, recipient_id, group_id,
		          body, state, created_at, updated_at`

	row := r.pool.QueryRow(ctx, query,
		msg.ClientID, msg.SenderID, msg.RecipientID, msg.GroupID, msg.Body, msg.State,
	)

	var out model.Message
	err := row.Scan(
		&out.ID, &out.ClientID, &out.SenderID, &out.RecipientID,
		&out.GroupID, &out.Body, &out.State, &out.CreatedAt, &out.UpdatedAt,
	)
	if err == pgx.ErrNoRows {
		// conflict -- fetch the existing row instead
		return r.GetByClientID(ctx, msg.ClientID)
	}
	if err != nil {
		return nil, fmt.Errorf("message insert: %w", err)
	}
	return &out, nil
}

// GetByID fetches a single message by its primary key.
func (r *MessageRepo) GetByID(ctx context.Context, id uuid.UUID) (*model.Message, error) {
	query := `
		SELECT id, client_id, sender_id, recipient_id, group_id,
		       body, state, created_at, updated_at
		FROM messages WHERE id = $1`

	var m model.Message
	err := r.pool.QueryRow(ctx, query, id).Scan(
		&m.ID, &m.ClientID, &m.SenderID, &m.RecipientID,
		&m.GroupID, &m.Body, &m.State, &m.CreatedAt, &m.UpdatedAt,
	)
	if err != nil {
		return nil, fmt.Errorf("message get by id: %w", err)
	}
	return &m, nil
}

// GetByClientID fetches a message by its client-side idempotency key.
func (r *MessageRepo) GetByClientID(ctx context.Context, clientID uuid.UUID) (*model.Message, error) {
	query := `
		SELECT id, client_id, sender_id, recipient_id, group_id,
		       body, state, created_at, updated_at
		FROM messages WHERE client_id = $1`

	var m model.Message
	err := r.pool.QueryRow(ctx, query, clientID).Scan(
		&m.ID, &m.ClientID, &m.SenderID, &m.RecipientID,
		&m.GroupID, &m.Body, &m.State, &m.CreatedAt, &m.UpdatedAt,
	)
	if err != nil {
		return nil, fmt.Errorf("message get by client_id: %w", err)
	}
	return &m, nil
}

// UpdateState sets the delivery state and bumps updated_at.
func (r *MessageRepo) UpdateState(ctx context.Context, id uuid.UUID, state model.DeliveryState) error {
	query := `UPDATE messages SET state = $1, updated_at = now() WHERE id = $2`
	tag, err := r.pool.Exec(ctx, query, state, id)
	if err != nil {
		return fmt.Errorf("message update state: %w", err)
	}
	if tag.RowsAffected() == 0 {
		return fmt.Errorf("message %s not found", id)
	}
	return nil
}
```

The idempotency pattern:
1. Try `INSERT ... ON CONFLICT (client_id) DO NOTHING RETURNING ...`
2. If the insert succeeds we get the row back via `RETURNING`.
3. If it conflicts, `RETURNING` returns zero rows (`pgx.ErrNoRows`). We fall back to a `SELECT` by `client_id`.

This means a client can safely retry the same message and will always get back the same server-assigned `id`.

## 5. Simple Client struct -- `internal/handler/client.go`


**`internal/handler/client.go`**

```go
package handler

import (
	"context"
	"encoding/json"
	"log"

	"github.com/google/uuid"
	"github.com/gorilla/websocket"

	"github.com/Rengoku1926/wp_proto/apps/backend/internal/model"
	"github.com/Rengoku1926/wp_proto/apps/backend/internal/repository"
)

// Client wraps a single WebSocket connection for one user.
type Client struct {
	conn     *websocket.Conn
	userID   uuid.UUID
	send     chan []byte
	msgRepo  *repository.MessageRepo
	registry *ConnRegistry // pointer back to the connection map
}

// readPump reads frames from the WebSocket and processes them.
// It runs in its own goroutine -- one per connection.
func (c *Client) readPump() {
	defer func() {
		c.registry.Remove(c.userID)
		c.conn.Close()
	}()

	for {
		_, raw, err := c.conn.ReadMessage()
		if err != nil {
			if websocket.IsUnexpectedCloseError(err,
				websocket.CloseGoingAway, websocket.CloseNormalClosure) {
				log.Printf("ws read error for %s: %v", c.userID, err)
			}
			return
		}

		var env model.Envelope
		if err := json.Unmarshal(raw, &env); err != nil {
			log.Printf("bad envelope from %s: %v", c.userID, err)
			continue
		}

		switch env.Type {
		case "message":
			c.handleMessage(env.Payload)
		default:
			log.Printf("unknown envelope type %q from %s", env.Type, c.userID)
		}
	}
}

// handleMessage processes an incoming "message" envelope.
func (c *Client) handleMessage(payload json.RawMessage) {
	var msg model.Message
	if err := json.Unmarshal(payload, &msg); err != nil {
		log.Printf("bad message payload from %s: %v", c.userID, err)
		return
	}

	// Force sender to the authenticated user (trust the connection, not the payload).
	msg.SenderID = c.userID
	msg.State = model.StatePending

	// Persist to Postgres.
	ctx := context.Background()
	saved, err := c.msgRepo.Create(ctx, &msg)
	if err != nil {
		log.Printf("failed to save message: %v", err)
		return
	}

	// Try to deliver to recipient if they are online.
	recipient := c.registry.Get(saved.RecipientID)
	if recipient != nil {
		// Wrap the saved message back into an envelope for the recipient.
		outPayload, _ := json.Marshal(saved)
		outEnv, _ := json.Marshal(model.Envelope{
			Type:    "message",
			Payload: outPayload,
		})

		select {
		case recipient.send <- outEnv:
			// Mark as sent since it reached the recipient's buffer.
			_ = c.msgRepo.UpdateState(ctx, saved.ID, model.StateSent)
		default:
			log.Printf("recipient %s send buffer full, dropping", saved.RecipientID)
		}
	}

	// Send ACK back to the sender.
	ackPayload, _ := json.Marshal(model.ACK{MessageID: saved.ID})
	ackEnv, _ := json.Marshal(model.Envelope{
		Type:    "ack",
		Payload: ackPayload,
	})
	select {
	case c.send <- ackEnv:
	default:
		log.Printf("sender %s send buffer full, dropping ack", c.userID)
	}
}

// writePump pumps messages from the send channel to the WebSocket.
// It runs in its own goroutine -- one per connection.
func (c *Client) writePump() {
	defer c.conn.Close()

	for msg := range c.send {
		if err := c.conn.WriteMessage(websocket.TextMessage, msg); err != nil {
			log.Printf("ws write error for %s: %v", c.userID, err)
			return
		}
	}
}
```

Each `Client` has two goroutines: `readPump` reads from the socket, `writePump` writes to it. They communicate through the `send` channel. This is the standard gorilla/websocket pattern.

Important: `msg.SenderID = c.userID` overwrites whatever the client sent. Never trust the client to identify itself -- use the connection's known identity.

## 6. Connection map -- `internal/handler/ws.go`

**`internal/handler/ws.go`**

```go
package handler

import (
	"log"
	"net/http"
	"sync"

	"github.com/google/uuid"
	"github.com/gorilla/websocket"

	"github.com/Rengoku1926/wp_proto/apps/backend/internal/repository"
)

// ConnRegistry is a thread-safe map of online user connections.
// This is the NAIVE version -- one connection per user, in-memory only.
// A later step replaces this with a Hub + Redis pub/sub.
type ConnRegistry struct {
	mu      sync.RWMutex
	clients map[uuid.UUID]*Client
}

func NewConnRegistry() *ConnRegistry {
	return &ConnRegistry{
		clients: make(map[uuid.UUID]*Client),
	}
}

func (r *ConnRegistry) Set(userID uuid.UUID, c *Client) {
	r.mu.Lock()
	defer r.mu.Unlock()
	// If there is an existing connection for this user, close it.
	if old, ok := r.clients[userID]; ok {
		close(old.send)
	}
	r.clients[userID] = c
}

func (r *ConnRegistry) Get(userID uuid.UUID) *Client {
	r.mu.RLock()
	defer r.mu.RUnlock()
	return r.clients[userID]
}

func (r *ConnRegistry) Remove(userID uuid.UUID) {
	r.mu.Lock()
	defer r.mu.Unlock()
	if c, ok := r.clients[userID]; ok {
		close(c.send)
		delete(r.clients, userID)
	}
}

// --- WebSocket HTTP handler ---

var upgrader = websocket.Upgrader{
	ReadBufferSize:  1024,
	WriteBufferSize: 1024,
	// Allow all origins for local dev. Lock this down in production.
	CheckOrigin: func(r *http.Request) bool { return true },
}

// WSHandler holds dependencies for the WebSocket endpoint.
type WSHandler struct {
	Registry *ConnRegistry
	MsgRepo  *repository.MessageRepo
}

func NewWSHandler(registry *ConnRegistry, msgRepo *repository.MessageRepo) *WSHandler {
	return &WSHandler{
		Registry: registry,
		MsgRepo:  msgRepo,
	}
}

// HandleWebSocket upgrades an HTTP request to a WebSocket connection.
// It expects a query parameter: ?user_id=<uuid>
//
// In a real app this would use a JWT or session token instead of a
// raw user_id in the query string. Authentication is added in a later step.
func (h *WSHandler) HandleWebSocket(w http.ResponseWriter, r *http.Request) {
	userIDStr := r.URL.Query().Get("user_id")
	if userIDStr == "" {
		http.Error(w, "missing user_id query param", http.StatusBadRequest)
		return
	}

	userID, err := uuid.Parse(userIDStr)
	if err != nil {
		http.Error(w, "invalid user_id: must be a UUID", http.StatusBadRequest)
		return
	}

	conn, err := upgrader.Upgrade(w, r, nil)
	if err != nil {
		log.Printf("ws upgrade failed: %v", err)
		return
	}

	client := &Client{
		conn:     conn,
		userID:   userID,
		send:     make(chan []byte, 256),
		msgRepo:  h.MsgRepo,
		registry: h.Registry,
	}

	h.Registry.Set(userID, client)

	// Start the read and write pumps in their own goroutines.
	go client.writePump()
	go client.readPump()

	log.Printf("user %s connected via WebSocket", userID)
}
```

Why a separate `ConnRegistry` struct instead of a bare `map`? Because in the next step we will swap this out for a Hub that also publishes to Redis. Wrapping it now means the `Client` code does not need to change.

## 7. Wire up the route -- `cmd/main.go`

Update your `cmd/main.go` to register the WebSocket endpoint. Add the highlighted lines:

```go
package main

import (
	"context"
	"net/http"
	"os"

	"github.com/Rengoku1926/wp_proto/apps/backend/internal/config"
	"github.com/Rengoku1926/wp_proto/apps/backend/internal/database"
	"github.com/Rengoku1926/wp_proto/apps/backend/internal/handler"
	"github.com/Rengoku1926/wp_proto/apps/backend/internal/logger"
	"github.com/Rengoku1926/wp_proto/apps/backend/internal/repository"
)

func main() {
	cfg, err := config.Load(".env.local")
	if err != nil {
		panic(err)
	}

	log := logger.New(cfg.Primary.Env)

	ctx := context.Background()

	pool, err := database.Connect(ctx, cfg.Database)
	if err != nil {
		log.Fatal().Err(err).Msg("failed to connect to database")
		os.Exit(1)
	}
	defer pool.Close()
	log.Info().Msg("successfully connected to postgres")

	if err := database.Migrate(ctx, pool); err != nil {
		log.Fatal().Err(err).Msg("failed to run migrations")
		os.Exit(1)
	}
	log.Info().Msg("migrations completed successfully")

	redisClient, err := database.ConnectRedis(ctx, cfg.Redis)
	if err != nil {
		log.Fatal().Err(err).Msg("failed to connect to redis")
		os.Exit(1)
	}
	defer redisClient.Close()
	log.Info().Msg("successfully connected to redis")

	// ---- NEW: set up WebSocket handler ----
	msgRepo := repository.NewMessageRepo(pool)
	registry := handler.NewConnRegistry()
	wsHandler := handler.NewWSHandler(registry, msgRepo)

	mux := http.NewServeMux()
	mux.HandleFunc("GET /ws", wsHandler.HandleWebSocket)

	log.Info().Msg("starting server on :8080")
	if err := http.ListenAndServe(":8080", mux); err != nil {
		log.Fatal().Err(err).Msg("server failed")
		os.Exit(1)
	}
}
```

You will also need to fetch the `google/uuid` package if you have not already:

```bash
go get github.com/google/uuid
```

## 8. Testing with wscat

### Install wscat

```bash
npm install -g wscat
```

### Start the server

```bash
go run cmd/main.go
```

You should see the migration logs and "starting server on :8080".

### Create two users

If you already have users from step 2, grab their UUIDs. Otherwise insert them directly:

```sql
INSERT INTO users (username) VALUES ('alice') RETURNING id;
-- e.g. a1b2c3d4-0000-0000-0000-000000000001

INSERT INTO users (username) VALUES ('bob') RETURNING id;
-- e.g. b1b2c3d4-0000-0000-0000-000000000002
```

### Connect as Alice (terminal 1)

```bash
wscat -c "ws://localhost:8080/ws?user_id=a1b2c3d4-0000-0000-0000-000000000001"
```

You should see `Connected`.

### Connect as Bob (terminal 2)

```bash
wscat -c "ws://localhost:8080/ws?user_id=b1b2c3d4-0000-0000-0000-000000000002"
```

### Alice sends a message to Bob

In Alice's terminal, paste:

```json
{"type":"message","payload":{"client_id":"11111111-1111-1111-1111-111111111111","recipient_id":"b1b2c3d4-0000-0000-0000-000000000002","body":"Hello Bob!"}}
```

**What you should see:**

- **Bob's terminal** receives:
  ```json
  {"type":"message","payload":{"id":"<server-uuid>","client_id":"11111111-1111-1111-1111-111111111111","sender_id":"a1b2c3d4-0000-0000-0000-000000000001","recipient_id":"b1b2c3d4-0000-0000-0000-000000000002","body":"Hello Bob!","state":1,...}}
  ```

- **Alice's terminal** receives an ACK:
  ```json
  {"type":"ack","payload":{"message_id":"<server-uuid>"}}
  ```

### Verify in Postgres

```sql
SELECT id, client_id, sender_id, recipient_id, body, state
FROM messages
WHERE client_id = '11111111-1111-1111-1111-111111111111';
```

You should see `state = 1` (SENT).

### Test idempotency

Paste the exact same message JSON in Alice's terminal again. Check Postgres -- there is still only one row. Alice gets the same ACK with the same `message_id`.
